# KN Win32 API Monitor v0.1.0.2

This patch tightens the release documentation and validation boundary around the broad runtime-monitoring pipeline introduced in v0.1.0.1.

## Highlights

- Clarified the current generated coverage model: `30,112` defined APIs and `30,092` runtime-monitorable APIs.
- Documented the split between compact Tier 0 transport IDs and the broader Tier 1/Tier 2 generic runtime profile path.
- Refreshed Tier 1, Tier 2, Tier 3, DLL batch promotion, and manual decoder planning numbers so the docs match the current generated artifacts.
- Added integration-test documentation for the API exerciser smoke, Tier 1/Tier 2 generic profile smokes, and generated generic preview smoke.
- Hardened generated-artifact line-ending rules so byte-for-byte validators stay stable on Windows checkouts.
